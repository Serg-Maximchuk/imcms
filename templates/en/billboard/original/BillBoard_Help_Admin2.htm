Help page for conference administrators

To create new HTML templates, there are a number of tags per template page which can be used.
Some of these tags can be used on all pages. These are listed below. After that list is another which lists all of the tags and
commands respectively which can be sent to the different servlets used in the conference.

General tags

Tag Meaning Example
#SERVLET_URL# Inserts path to servlet. http://demo.imcode.com/servlet/
#IMAGE_URL# Inserts path to conference image folder http://demo.imcode.com/images/

General parameters
Some of the parameters use servlets now and then. These are taken primarily from the request object and if they are not found there, then they are taken from the session object.

Parameters Meaning
META_ID Indicates the meta ID of the conference
PARENT_META_ID Indicates the meta ID from which "our" meta ID has inherited its properties
COOKIE_ID Not used in this module.
FORUM_ID Indicates the forum in point
DISC_ID Indicates the discussion in point
LAST_LOGIN_DATE Indicates the most recent date the user was logged into the conference
USER_ID Indicates which users are logged into the conference
DISC_INDEX Indicates which value the discussion index has. In other words which discussion is being used as a reference for calculating which discussion shall be shown to the user.

 

ConfAdd servlet
Post-parameters: ConfAdd servlet supports the following parameters.

Parameters Meaning Value
CANCEL Indicates that the user wishes to stop the creation of a contribution/discussion Arbitrary
ADD Indicates that the user wishes to add a new discussion/contribution Arbitrary

Add parameters: If the function parameter is add, the following parameters are required:

ADD_HEADER Inserts the header of the contribution Arbitrary
ADD_TEXT Inserts the text of the contribution Arbitrary
ADDTYPE Inserts the type of the contribution that is made, either REPLY contribution or an initial DISCUSSION contribution REPLY, DISCUSSION

Tags: ConfAdd servlet supports the following tags.

Tag Meaning
#FIRST_NAMEL# Inserts the author's given name
#LAST_NAMEL# Inserts the author's family name
#ADD_TYPE# Inserts the type of contribution (DISCUSSION/REPLY)
#DISC_HEADER# Inserts the current discussion header

ConfAdmin servlet
Tags: The following tags are supported.

Tag Meaning Get-parameter
#EXTERNAL_PATH# Used in admin mode to indicate where an uploaded file should be placed on a server. META
#TEMPLATE_LIST# Indicates a list of templates found in a template directory META
#A_META_ID# Indicates the meta ID of the active conference har META
#CURRENT_TEMPLATE_SET# Used to show which template directory is in use META
#FORUM_LIST# Indicates a list of those forums found in a conference FORUM
#NBR_OF_DISCS_TO_SHOW_LIST# Indicates a special form of forum list with contains not only forum name but the number of discussions shown as well FORUM
#NEW_A_HREF_LIST# Inserts all new discussions created since an administrator was last logged in DISCUSSION
#OLD_A_HREF_LIST# Inserts all old discussions since admin was last logged in DISCUSSION
#USER_SORT_ORDER# Inserts the user's sorting order of contributions. Can be 1 or 0. REPLY
#CHECKBOX_STATE# Inserts the word checked / or nothing depending on the user's sorting order. REPLY
#REPLIES_RECORDS# Inserts a list with all contributions to a discussion REPLY

#CONF_UNADMIN_BUTTON#

Inserts image for a button to end administration

#CONF_UNADMIN_LINK#

Inserts button to end administration

Post-parameters:

Parameter Meaning parameter value
REGISTER_TEMPLATE_LIB Register a new template directory Arbitrary
UPLOAD_CONF Indicates the name of the first forum Arbitrary
SET_TEMPLATE_LIB Indicates which template directory should be used in a discussion Arbitrary
DELETE_REPLY Deletes a contribution in a discussion Arbitrary
RESAVE_REPLY Resaves an edited contribution Arbitrary
DELETE_DISCUSSION Deletes a discussion Arbitraryt
DELETE_FORUM Deletes a forum Arbitrary
ADD_FORUM Adds a forum Arbitrary
CHANGE_FORUM_NAME Changes the name of an existing forum Arbitrary
SHOW_DISCUSSION_NBR Sets the number of discusssions which should shown in a forum Arbitrary

REGISTER_TEMPLATE_LIB parameters: If the function parameter is REGISTER_TEMPLATE_LIB the following parameters are required.

TEMPLATE_LIB_NAME Indicates the new name of the new template directory Arbitrary

UPLOAD_CONF parameters: If function parameter is UPLOAD_CONF the following parameters are required.

TEMPLATE_NAME Indicates the name of the template directory where the upload should be executed Existing template directory (selected from the list)
UPLOAD_TYPE Indicates the file type being uploaded. IMAGE/TEMPLATE

SET_TEMPLATE_LIB parameters: If function parameter is SET_TEMPLATE_LIB the following parameters are required.

TEMPLATE_ID Indicates the meta ID of the new template directory selected Existing template directory (selected from the list)

RESAVE_REPLY parameters: If function parameter is RESAVE_REPLY the following parameters are required.

TEXT_BOX_x Indicates the text box ID of a contribution. Example: TEXT_BOX_2 // Indicates text box no. 2
REPLY_HEADER_x Indicates the reply header ID of a contribution. Exempel: REPLY_HEADER_2 // Indicates the header of contribution no. 2

ConfDisc servlet
Tags: The following tags are supported.

Tag Meaning
#A_HREF_LIST# Indicates a list of all discussions in a forum. Comes as a href tag

#PREVIOUS_BUTTON#

Inserts button to list previous discussions

#NEXT_BUTTON#

Inserts button to list later discussions

#NEW_DISC_BUTTON#

Inserts button to add discussions

#CONF_ADMIN_BUTTON#

Inserts image for administration button

#CONF_ADMIN_LINK#

Inserts button for administration of discussions

Get-parameters: None
Post-parameters:

Parameter Event parameter value
UPDATE Updates the discussion servlet Arbitrary
NEXT Generates a new discussion list with x number of latest created discussions if such exists. Arbitrary
PREVIOUS Generates a new discussion list with x number of earlier created discussions if such exists. Arbitrary
SEARCH Generates a new discussion list with all hits made with a search Arbitrary

Search parameters:
The following parameters will be necessary if SEARCH has been sent to ConfDisc Post function.

Search parameter Meaning parameter value
CATEGORY Indicates in which category the search should be executed. 0=headers, 1=texts, 2=users 0/1/2
SEARCH Indicates search word(s) which should be searched for Arbitrary
FR_DATE Indicates the start date for the search frame Arbitrary
FR_VALUE Indicated the default value to be used in the date field. In English, this is yesterday. FR_VALUE should agree with the FR_DATE default value Arbitrary
TO_DATE Indicates the end date for the search frame Arbitrary
TO_VALUE Indicated the default value to be used in the date field. In English, this is today. TO_VALUE should agree with the TO_DATE default value Arbitrary

ConfLogin servlet
Tags: The following tags are supported. Get takes two parameters, if neither of these parameters is entered, generates log in page automatically.

Tag Meaning
#USERS_MENU# Inserts an option list with users in conference

Get-parameters:

Parameter Event parameter value
login_type Generates HTML page for administration of users ADMIN_USER
login_type Generates HTML page for adding new users ADD_USER
Generates HTML page for logging in none (generates if none of the above are entered)

Post-parameters:

Parameter Event parameter value/b>
login_type Executes if login_type AND submit have be sent. Verifies log-in to conference. LOGIN
login_type Adds a user into imCMS user database and to the conference's database ADD_USER
login_type Saves a user's user-level to conference database SAVE_USER
